### 📊 Logic & Analytics
This directory houses the JavaScript responsible for rendering our **Light Status Charts**, **Users and Usage Charts** and **Data Graphs**. We turn boring numbers into visual constellations!

### 🧠 The Brain of the Dashboard
This folder contains `dashboard2.js`, which handles all the interactive data visualization for the project. It uses **Chart.js** to render night-themed analytics.

### 🔭 Features Inside `dashboard2.js`
* **Revenue Analytics:** A bar chart with a dynamic **Year Selector** (2024/2025) that updates in real-time.
* **Growth Tracking:** Visualizes monthly user sign-ups.
* **System Status:** Pie and Donut charts for **Role Distribution** and **Light System** health.
* **Performance:** All charts are set to `maintainAspectRatio: false` for perfect responsive scaling.

### 🎨 The "Gold Standard" Theme
To match our site's aesthetic, all charts use:
- **Primary Glow:** `rgba(255, 215, 0, 0.6)` (Gold)
- **Grid Lines:** Soft white transparency `rgba(255, 255, 255, 0.1)`
- **Typography:** Bold white labels for high readability on dark backgrounds.

---

### 💡 Data Tip
> "If a graph looks too crowded on a phone, remember to use `responsive: true` in your chart settings so the stars don't collide!"

### 🛠️ Quick Maintenance
If you add a new year of data:
1. Open `dashboard2.js`.
2. Update the `years` array: `const years = [2024, 2025, 2026];`.
3. Add the new data object to `revenueData`.
